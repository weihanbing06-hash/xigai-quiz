# 习概刷题小程序（第一代）

## 启动

双击 `index.html`，使用 Edge、Chrome 等现代浏览器打开即可。程序完全离线运行。

请不要单独移动 `index.html`。`index.html`、`app.js`、`styles.css` 和 `question-bank.js` 需要始终放在同一文件夹内。

## 键盘操作

- 单选题：按 `1`、`2`、`3`、`4` 直接作答并提交。
- 判断题：按 `1` 选择“对”，按 `0` 选择“错”；是否自动提交由设置决定。
- 多选题：依次按数字选择或取消选项，按 `Enter` 提交。
- 显示判题结果后：使用设置中的“下一题”快捷键继续。
- 背题模式：使用设置中的“上一题”“下一题”快捷键翻页。

## 手机与触屏操作

- 每种答题模式底部都有“上一题”和“下一题”按钮。
- 在题号框输入数字并点击“跳转”，可以直接前往指定题目。
- 多选题勾选完成后，点击蓝色“提交答案”按钮。
- 模拟考试允许前后检查题目，已选答案会保留；最后一题会提示交卷或自动寻找未答题。

## 答题设置

页面右上角“设置”中可调整：

- 单选、判断自动提交：默认开启；关闭后选择答案，再点击“提交答案”判题。
- 答对后自动进入下一题：默认关闭；开启后会先显示约 0.9 秒正确反馈，再自动切题。答错时不会自动跳转。
- 自定义按键：可分别设置选项1—4、判断题对/错、提交答案、上一题和下一题。点击按键框后直接按下新键，保存后生效。
- 模拟考试按题型分区排列：第 1—20 题为单选题，第 21—30 题为多选题，第 31—40 题为判断题；各题型内部随机抽题。
- 模拟考试限时 20 分钟，时间归零后自动交卷，未作答题目按错误计入。
- 单选题和多选题的选项会在每次开始学习时随机排列；同一次学习中返回或跳转时顺序保持不变。判断题仍固定使用 `1=对、0=错`。

## 学习状态与错题规则

- 初始时 969 道题均标记为“未学”，错题本为空。
- 背题模式不改变掌握状态。
- 学习、错题复习和模拟考试都会更新掌握状态。
- 首次作答后，尚未连续正确两次的题标记为“学习中”。
- 连续答对两次：标记为“已掌握”；若在错题本中，同时移出错题本。
- 任意一次答错：连续正确次数归零，标记为“错题”并加入错题本。
- 已掌握题以后再次答错，也会降回错题状态并重新进入错题本。

## 学习模式

- 主线新题按照章节顺序推进，每学习约 2—4 道新题，会穿插一道需要复习的旧题。
- 未掌握题、历史错题优先出现；已掌握题只会低频抽查。
- 刚答错的题会间隔若干题后再次出现，不会立即连续轰炸。
- 已掌握题在任何模式中再次答错，都会立即重新进入错题本。
- 学习过程不设置固定轮次，可随时退出，之后从上次位置继续。
- 主线新题全部完成后，学习模式会继续以未掌握题和历史错题为主进行巩固。

## 数据保存

学习记录会自动保存在当前浏览器中。建议每天结束时点击右上角“导出进度”备份。更换浏览器、清理浏览器数据或更换电脑时，可以使用“导入进度”恢复。

进行中的背题、刷题、错题复习和模拟考试也会自动保存。误关页面或刷新后，可在首页点击“继续学习”恢复题号、答案、随机选项顺序与考试剩余时间。

## 模拟考试答题卡

- 考试页面顶部提供“答题卡”和“交卷”按钮。
- 答题卡按单选、多选、判断分区显示，蓝色代表已答，红色边框代表当前题。
- 可点击任意题号直接跳转。
- 主动交卷时会提示未答题数量；未答题按错误计入。

## 重新生成题库

如果原 Markdown 题库有修改，在本文件夹打开终端并运行：

```powershell
node .\build-question-bank.mjs
```

生成程序会核对题目总数和三种题型数量，校验通过后更新 `question-bank.js`。
